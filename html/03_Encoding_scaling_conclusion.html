Codificacin del resto de variables categoricas

Existen varias opciones para el encoding, podriamos codificarlas todas con one-hot-encoding, o tambien se podra elegir codificar algunas de una forma y otras de otra. La primera idea era codificar las variables con pocas categoras con one-hot-encoding y variables con ms categorias con Target Encoding

Elejimos codificar las variables categricas tipo string de las siguiente maneras:

  • Si tenemos variables con 2 categoras : las pasamos manualmente a tipo booleano (0,1).

  • Si tenemos variables con 3 o menos categorias : utilizamos One-Hot Encoding.

  • Si tenemos variables que parezcan tener un orden categrico ascendente : utilizamos Ordinal Encoding.

  • Si tenemos variables con ms de 3 categorias : codificamos mediante Target Encoding.

One-Hot Encoding es ideal para pocas categoras, ya que evita suposiciones sobre relaciones, mientras que Target Encoding es ms eficiente para muchas categoras, al reducir dimensionalidad y capturar patrones relevantes con la variable objetivo.

Pero como solo tenemos 2 variabless con 3 o menos categorias, hemos decidido hacer la que tiene 2 categorias manualmente, y la que tiene 3 categoras nos cuadra ms utilizando ordinal Encodding porque sigue un orden categrico ascendente.
